变量

var是关键词，也可以用val代替。

注意事项：

  1. 声明变量时，类型可以省略；

  2. 类型确定后不能更改，因为Scala是强类型语言;

  3. 通过var声明的变量值可以修改，通过val声明的不可以修改；

    设计var和val的原因

    1. 实际编程中很多时候是为了获取对象属性或修改属性值，而很少修改对象本身，此时可以使用val；
    2. val没有线程安全问题所以效率高，推荐尽量使用val；
    3. 如果需要改变，使用var；

  4. val修饰的变量在编译后，等同于加上final；

  5. 变量声明时需要初始值；

Scala中的加法运算

  1. 如果两边都是数值型，则做加法运算；
  2. 如果有一侧是字符串类型，则做拼接操作；

Scala中的数据类型

  1. Scala中数据类型都是对象；

  2. Scala中数据类型有两中，AnyVal（值类型）和AnyRef（引用类型），Null是AnyRef类型的底层类，Nothing是所有类型的底层类；

    1. Any类型是所有类的父类；

    2. Null类型是特别的类型，只有一个值null，是底层类，是所有AnyRef类型的子类；

    3. Nothing类型也是特别的类型，是所有类的子类，在开发中可以将Nothing类型的值返回给任意变量或函数，在抛出异常中使用很多；

    4. 仍然遵守低精度值向高精度值自动隐式转换，高精度转换为低精度需要显式类型转换；

值类型转换

隐式类型转换

进行赋值运算时，精度小的类型可以自动转换为精度大的类型，这就是自动类型转换（隐式转换）。

Byte->Short->Int(Char->Int)->Long->Float->Double

细节：

  1. 多种数据类型进行混合运算时，系统首先将所有数据类型转换成容量最大的那种再计算；
  2. 精度大的数据类型赋值给精度小的数据类型就会报错，需要强制类型转换；
  3. Byte和Short与Char之间不会相互自动转换；
  4. Byte和Short与Char在计算时首先转换为Int；
  5. 表达式结果的类型自动提升为操作数中最大的类型；

高级隐式转换机制以后再讨论。

强制类型转换

将精度高的数据类型转换为容量小的数据类型，需要使用强制类型转换，可能会造成精度降低或溢出。

细节：

  1. 数据容量和精度从大到小就需要强制转换；
  2. 强制转换符号只针对最近的操作数有效，可以用小括号来作用于更大范围；
  3. Char类型可以报错Int的常量值，但不能保存Int的变量值；
  4. Byte和Short类型在进行计算时，当做Int类型处理；

值类型和String类型之间的转换

  1. 基本转String类型直接用+即可；
  2. String转基本使用toXXX方法；
  3. String转基本时要确保能转成有效的数据，比如“Hello”就不能转成整数；
  4. 浮点数通过toInt转成整数也会出错；

标识符

对各种变量、方法、函数等命名使用的字符序列称为标识符，简单地说就是自己起名字的地方。

命名规则与Java基本一致：

  1. 首字符为字母或下划线，后续字符任意字母和数字、美元符号，可以后接下划线_；

  2. 数字不可以开头；

  3. 首字符为操作符（比如+ - * /）时，后续字符也需要跟操作符，至少一个；

  4. 操作符不能在标识符中间或最后；

  5. 用反引号` `包裹的任意字符串也可以为标识符，关键字也可以；

  6. Int等可以作为标识符，因为他们不是关键字，而是预定义标识符，可以用但是不推荐；

  7. $符号在Scala中也看作字符，但$开头的标识符为保留的编译器产生的标识符，应该避免使用防止冲突；

  8. 单独的下划线有其他作用，不能作为标识符；

命名规范：

  1. 包名：尽量采取有意义的包名，简短有意义；（com.公司名.项目名.模块名）
  2. 变量名、函数名、方法名采用驼峰法；
